Open the Firefox Accessibility Panel or the "Landmarks" Chrome extension to see the landmark tree.

← Back to overview
Screen reader shortcuts:
VoiceOver macOS: Ctrl + Opt + U → Rotor → Landmarks
NVDA Windows: D = next landmark  ·  Shift + D = previous
Firefox: Tools → Browser Tools → Accessibility → look for "landmark" roles in the tree

Each dashed border shows one landmark region — purely visual annotation for the presentation. In a real page there would be no borders.

The markup

<header>
  <nav aria-label="Main navigation">…</nav>
</header>
<main>
  <section aria-labelledby="news-title">
    <h2 id="news-title">Latest articles</h2>
  </section>
</main>
<aside aria-label="Related topics">…</aside>
<footer>
  <nav aria-label="Footer navigation">…</nav>
</footer>

Two <nav> elements on one page: both need an aria-label to distinguish them. Without it, a screen reader announces "Navigation" twice with no way to tell them apart.
