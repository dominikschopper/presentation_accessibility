Toast notification — aria-live="polite"

Bad — visual only, AT is silent

Toast appears — screen reader says nothing.

<div class="toast" v-if="saved">
  Saved!
</div>
<!-- no live region -->
Good — live region announces message

Toast appears and screen reader announces "Document saved!" politely.

<!-- Always in DOM, never v-if -->
<p role="status"
   aria-live="polite"
   aria-atomic="true"
   class="sr-only">
  {{ message }}
</p>

aria-atomic — upload progress

Without aria-atomic, the screen reader reads only the changed text node — just the number, no context.

atomic="false" — only the change is read
0 of 10 files uploaded

AT reads: "4" … "5" … — no context

atomic="true" — full sentence is read
0 of 10 files uploaded

AT reads: "4 of 10 files uploaded" — full context

aria-live="assertive" — critical errors only

Assertive regions interrupt the screen reader immediately. Use only for urgent, critical errors.

<div role="alert" aria-live="assertive">
  {{ criticalError }}
</div>
