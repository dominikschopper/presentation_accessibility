The trap — aria-hidden on a focusable element

Focusable elements inside an aria-hidden container are still reachable by Tab — but the screen reader won't announce them. Focus disappears into a void.

Bad — button hidden from AT but reachable

aria-hidden="true" on this container

  <button>Dangerous!</button>
</div>

The sr-only pattern

.sr-only {
  position: absolute;
  width: 1px; height: 1px;
  padding: 0; margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

Visually invisible, but present in the accessibility tree. Never use display:none or visibility:hidden — those remove the element from the AT entirely.
