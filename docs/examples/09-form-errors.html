09 — Form Errors

Submit each form empty and listen to what the screen reader announces.

← Back to overview

Bad — errors visible, AT uninformed

Errors appear visually but focus stays on the button. Screen reader users have no indication anything went wrong.

Name is required.

Please enter a valid email address.

<p class="error" v-if="error">
  Name is required.
</p>
<!-- No aria-invalid, no role="alert",
     no focus management -->

Good — errors announced, focus moved

On submit: a summary appears with role="alert", and focus moves to the first invalid field.

Name is required.

Please enter a valid email address.

<div role="alert" aria-live="assertive">
  {{ errorSummary }}
</div>
<input
  aria-required="true"
  aria-describedby="name-err"
  :aria-invalid="nameError ? 'true' : undefined"
/>
