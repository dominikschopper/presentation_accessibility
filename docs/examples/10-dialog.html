10 — Modal Dialog

Open each dialog and navigate with Tab and ESC. Notice the difference in focus trap and background accessibility.

← Back to overview
What to check: After opening — can you Tab outside the dialog? Does ESC close it? After closing — does focus return to the button that opened it? Is the background content reachable by Tab?

Bad — div "modal", no native behaviour

Built with a div. No focus trap, no ESC support, background remains accessible by Tab and screen reader.

Good — native <dialog> with inert

Native <dialog> with showModal(). Focus trap and ESC are built in. Background is made inert.

  • Focus trap — Tab stays within the dialog
  • ESC to close — built into the browser
  • Background inert — background links unreachable
  • Focus returns to trigger — on close()
<dialog
  aria-labelledby="title"
  aria-describedby="desc">
  …
</dialog>

// JS
dialog.showModal()   // activates modal mode
main.inert = true    // blocks background
// on close:
main.inert = false
trigger.focus()      // return focus

Why inert and not aria-hidden?

aria-hidden="true"
  • Hides from the accessibility tree ✓
  • Tab focus still works through it ✗
  • Mouse clicks still work ✗
inert
  • Hides from the accessibility tree ✓
  • Blocks all Tab focus ✓
  • Blocks mouse clicks ✓

Browser support: Chrome 102+, Firefox 112+, Safari 15.5+. If you need to support older browsers, use the wicg-inert polyfill.
