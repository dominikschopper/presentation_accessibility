The trap: aria-hidden on a focusable element

If a focusable element is inside an aria-hidden container, keyboard users can still Tab to it — but the screen reader will not announce it. The focus disappears into a void.

The sr-only pattern

.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

Visually invisible, but present in the accessibility tree. Do not use display:none or visibility:hidden — those remove the element from the AT as well.
