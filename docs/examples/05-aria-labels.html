05 — ARIA Labels

Tab to each button and listen to what the screen reader announces.

← Back to overview

Icon buttons

Bad — no label

Screen reader: "button" — no idea what any of these do.

<button>💾</button>
<button>🗑</button>
<button>✎</button>
Good — aria-label

Screen reader: "Save document, button" — clear and unambiguous.

<button aria-label="Save document">
  <span aria-hidden="true">💾</span>
</button>

aria-labelledby — referencing visible text

When a visible heading already describes a region, reference it instead of repeating the text.

Account settings

This <section> is labelled by its heading via aria-labelledby="settings-heading". Screen reader: "Account settings, region".

<section aria-labelledby="settings-heading">
  <h3 id="settings-heading">Account settings</h3>
  …
</section>

aria-describedby — supplementary description

A label names the element. A description adds context. Both are read on focus — label first, description second.

Bad — hint not associated

At least 8 characters, 1 number, 1 special character

<input id="pw" type="password" />
<p>At least 8 characters…</p>
Good — aria-describedby

At least 8 characters, 1 number, 1 special character

Screen reader on focus: "Password, edit text. At least 8 characters, 1 number, 1 special character."

<input id="pw" type="password"
  aria-describedby="pw-hint" />
<p id="pw-hint">At least 8…</p>
