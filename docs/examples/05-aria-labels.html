Tab to each button and listen to what the screen reader announces.

Icon buttons

Bad — no label

Tab to these buttons. The screen reader announces only "button" — no idea what they do.

<button>💾</button>
<button>🗑</button>
<button>✎</button>
Good — aria-label

Tab to these buttons. The screen reader announces the action clearly.

<button aria-label="Save document">
  <span aria-hidden="true">💾</span>
</button>

aria-labelledby — referencing visible text

When a visible heading already describes a region, reference it instead of duplicating the text in an aria-label.

Account settings

This <section> is labelled by its visible heading via aria-labelledby="settings-heading". In the accessibility tree this region is announced as "Account settings, region".

<section aria-labelledby="settings-heading">
  <h3 id="settings-heading">Account settings</h3>
  <p>…</p>
  <button>Save changes</button>
</section>

aria-describedby — supplementary description

A label names the element. A description adds context. Both are read by the screen reader when the element receives focus — first the label, then the description.

Bad — no description

The hint exists visually, but is not associated with the input.

At least 8 characters, 1 number, 1 special character

<label for="pw">Password</label>
<input id="pw" type="password" />
<p>At least 8 characters…</p>
Good — aria-describedby

At least 8 characters, 1 number, 1 special character

Screen reader on focus: "Password, edit text. At least 8 characters, 1 number, 1 special character."

<label for="pw">Password</label>
<input id="pw" type="password"
  aria-describedby="pw-hint" />
<p id="pw-hint">At least 8…</p>
