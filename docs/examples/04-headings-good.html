04 — Heading Hierarchy (Good)

Strict nesting — visual size controlled via CSS, not by choosing a different heading element.

← Back to overview
Try it: Use H (NVDA) or the Rotor (VoiceOver) to jump between headings. The outline reads like a logical table of contents — no gaps, no surprises.
Good — correct heading hierarchy

Company Blog h1

Featured article h2

Proper h2, one level below the page h1. Visual size is set via CSS.

Author information h3

Written by Jane Doe · 12 January 2025

More articles h2

A sibling of "Featured article" — both are direct children of the h1.

Newsletter signup h3

Correctly appears as h3 under the h2. No levels skipped.

What a screen reader user sees

h1  Company Blog
  h2  Featured article
    h3  Author information
  h2  More articles
    h3  Newsletter signup

Every level follows naturally from the previous one. Compare with 03 — Heading Hierarchy (Bad).

Visual size without breaking semantics

.card h2 { font-size: 1rem; font-weight: 600; }
