Featured article h2

This is a proper h2, one level below the page h1. Visual size is set via CSS, not by picking a higher heading level.

More articles h2

A second h2, a sibling of "Featured article". Both are direct children of the h1. The hierarchy is clear.

Newsletter signup h3

This subsection now correctly appears as h3 under the h2. No levels were skipped.

What a screen reader user hears

When navigating by heading, the outline of this page looks like this:

h1  Company Blog
  h2  Featured article
    h3  Author information
  h2  More articles
    h3  Newsletter signup

Every level follows naturally from the previous one. A screen reader user can understand the page structure without seeing it.

Controlling visual size without breaking semantics

/* Use CSS to make an h2 look smaller if needed */
.card h2 {
  font-size: 1rem;   /* visually smaller */
  font-weight: 600;
}
