Toast notification — aria-live="polite"

A toast appears visually and is simultaneously announced by the screen reader — without moving keyboard focus.

Bad — visual only, AT is silent

The toast appears — but there is no live region. The screen reader says nothing.

Document saved!
<div class="toast" v-if="saved">
  Saved!
</div>
Good — live region announces the message

The toast appears and the screen reader announces "Document saved!" without interrupting the user.

<!-- Always in the DOM, never v-if -->
<p
  role="status"
  aria-live="polite"
  aria-atomic="true"
  class="sr-only"
>{{ message }}</p>

aria-atomic — upload progress

Compare how aria-atomic="false" and aria-atomic="true" sound when a counter updates.

atomic="false" — only the change is read

When the number updates, the AT reads only the changed text node — just the number, no context.

0 of 10 files uploaded
atomic="true" — full text is read

When the number updates, the AT reads the entire text — "4 of 10 files uploaded" — with full context.

0 of 10 files uploaded

aria-live="assertive" — critical errors only

Assertive regions interrupt the screen reader immediately. Use only for urgent errors — not for status updates or confirmations.

<div role="alert" aria-live="assertive">
  {{ criticalError }}
</div>
